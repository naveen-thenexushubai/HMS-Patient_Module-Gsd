<objective>
Close REG-12 gap: enforce photo ID verification at registration time so the system requires `photoIdVerified=true` before a patient can be registered.

Purpose: REG-12 states "System requires photo ID verification (scan or upload) during registration". Currently the `photo_id_verified` DB column exists with DEFAULT false, the `Patient` entity has the field, but `RegisterPatientRequest` has no `photoIdVerified` field and the API allows registration without any photo ID check. This plan adds the field to the request DTO and enforces it via Bean Validation — the simplest approach for a backend-only Phase 1 system with no file storage. The actual scan/upload UI is a Phase 3 concern. The API-level flag enforcement satisfies REG-12 for Phase 1.

Output: Modified RegisterPatientRequest DTO with photoIdVerified field and @AssertTrue validation. PatientService updated to pass the field to the Patient builder. Integration tests for the 400/201 boundary.
</objective>

**Step 1 — Add photoIdVerified to RegisterPatientRequest.**

Add the following field to `RegisterPatientRequest.java` after the `zipCode` field and before the `emergencyContacts` field:

```java
@NotNull(message = "Photo ID verification is required")
@AssertTrue(message = "Photo ID must be verified before registration")
private Boolean photoIdVerified;
```

The `@AssertTrue` annotation on a `Boolean` field causes Bean Validation to reject `false` as invalid (only `true` passes). Combined with `@NotNull`, this means:
- `null` → 400 with fieldError: `photoIdVerified: "Photo ID verification is required"`
- `false` → 400 with fieldError: `photoIdVerified: "Photo ID must be verified before registration"`
- `true` → passes validation, registration proceeds

The import `jakarta.validation.constraints.AssertTrue` must be added if not already present. Other imports (`NotNull`, etc.) already exist.

**Step 2 — Pass photoIdVerified to Patient builder in PatientService.**

In `PatientService.registerPatient()`, the `Patient.builder()` block currently does not set `photoIdVerified`. Add:
```java

```

This line should be added after `.zipCode(request.getZipCode())` and before `.status(PatientStatus.ACTIVE)` in the builder chain.

The `Patient` entity already has `private Boolean photoIdVerified = false;` (line 87-88 in Patient.java) and the DB column `photo_id_verified BOOLEAN DEFAULT false` exists. No schema changes needed.

Add two new integration test methods to `Phase01VerificationTest` that verify the `photoIdVerified` enforcement behavior. These tests go AFTER the existing 8 test methods.

```java
/**
 * REG-12 GAP CLOSURE:
 * Registration must be rejected (400 Bad Request) when photoIdVerified is false.
 * Verifies that the @AssertTrue constraint fires and the RFC 7807 response
 * contains the correct fieldErrors entry.
 */
@Test
@WithMockUser(roles = "RECEPTIONIST")
void reg12_registrationRejectedWhenPhotoIdNotVerified() throws Exception {
    RegisterPatientRequest request = RegisterPatientRequest.builder()
        .firstName("Test")
        .lastName("Patient")
        .dateOfBirth(LocalDate.of(1990, 1, 1))
        .gender(Gender.MALE)
        .phoneNumber("555-123-9999")
        .email("test.patient@example.com")
        .photoIdVerified(false)  // Explicitly not verified
        .build();

    mockMvc.perform(post("/api/v1/patients")
            .contentType(MediaType.APPLICATION_JSON)
            .content(objectMapper.writeValueAsString(request)))
        .andExpect(status().isBadRequest())
        .andExpect(jsonPath("$.status").value(400))
        .andExpect(jsonPath("$.fieldErrors").isMap())
        .andExpect(jsonPath("$.fieldErrors.photoIdVerified").exists());
}

/**
 * REG-12 GAP CLOSURE:
 * Registration succeeds (201 Created) when photoIdVerified is true.
 * Verifies the happy path for the photo ID verification flag.
 */
@Test
@WithMockUser(roles = "RECEPTIONIST")
void reg12_registrationSucceedsWhenPhotoIdVerified() throws Exception {
    RegisterPatientRequest request = RegisterPatientRequest.builder()
        .firstName("Verified")
        .lastName("Patient")
        .dateOfBirth(LocalDate.of(1988, 6, 15))
        .gender(Gender.FEMALE)
        .phoneNumber("555-456-7890")
        .email("verified.patient@example.com")
        .photoIdVerified(true)  // Verified — registration allowed
        .build();

    mockMvc.perform(post("/api/v1/patients")
            .contentType(MediaType.APPLICATION_JSON)
            .content(objectMapper.writeValueAsString(request)))
        .andExpect(status().isCreated())
        .andExpect(jsonPath("$.patientId").exists())
        .andExpect(jsonPath("$.firstName").value("Verified"));
}
```
