Create comprehensive HIPAA Security Risk Assessment documenting all PHI storage and transmission paths for Phase 0.

**Structure (based on HHS guidance):**

1. **Executive Summary**
   - Scope: Patient Management System - Phase 0 Security Foundation
   - Assessment date, version, responsible party

2. **PHI Inventory**
   - Database: PostgreSQL tables storing patient demographics (name, DOB, address, phone, email, SSN, insurance)
   - API endpoints: REST endpoints exposing patient data
   - Logs: Application logs and audit logs containing PHI
   - Backups: Database backups containing PHI

3. **Technical Safeguards Implemented (Phase 0)**
   - SEC-02: Audit logging with pgAudit + application-level tracking (6-year retention)
   - SEC-03: Data at rest encryption (PostgreSQL TDE or OS-level encryption - document choice)
   - SEC-04: Data in transit encryption (TLS 1.3 on all API communications)
   - SEC-05: Field-level encryption for SSN and insurance policy numbers (JPA AttributeConverter with AES-256)
   - SEC-06: JWT authentication with role-based access control (ADMIN, DOCTOR, NURSE, RECEPTIONIST)
   - SEC-07: Object-level authorization with custom PermissionEvaluator
   - SEC-08: Secrets stored in environment variables (documented migration path to Vault)
   - SEC-09: Audit logs capture user ID, timestamp, action, resource ID, IP address, user agent

4. **Threat Analysis**
   - Unauthorized access to patient records (mitigated by JWT + object-level authorization)
   - Data breach via unencrypted storage (mitigated by encryption at rest + field-level encryption)
   - Man-in-the-middle attacks (mitigated by TLS 1.3)
   - Insider threats (mitigated by audit logging + least privilege access)
   - Exposed secrets (mitigated by environment variables + .gitignore)
   - Actuator endpoint exposure (mitigated by disabling all except /health and /info)

5. **Vulnerability Assessment**
   - CVE-2025-22235: Addressed by Spring Boot 3.4.5+
   - Weak JWT secrets: Addressed by mandatory 64-character base64 secrets
   - Missing object-level authorization: Addressed by PermissionEvaluator pattern

6. **Risk Mitigation Status**
   - List all 9 SEC requirements with implementation status
   - Note open questions from research (secrets manager choice, database encryption strategy)

7. **Compliance Gaps**
   - Secrets management: Currently environment variables, recommend HashiCorp Vault or AWS Secrets Manager for production
   - Audit log archival: PostgreSQL partitioning implemented, cold storage strategy TBD based on volume
   - Authorization rules: Placeholder PermissionEvaluator in Phase 0, refined rules in Phase 1

8. **Recommendations**
   - Phase 1: Integrate HashiCorp Vault or AWS Secrets Manager
   - Phase 1: Define detailed object-level authorization rules based on patient-provider assignments
   - Phase 1: Implement audit log monitoring dashboard
   - Phase 2: Implement automated backup encryption and testing

Document must be specific to this application, not generic HIPAA checklist. Reference actual file paths, configuration settings, and architectural decisions from Plans 02-04.
